# 2. 接入指引

# 2.1 接口协议

服务调用提供HTTP和HTTPS两种协议方式，支持GET和POST请求方式。

# 2.2 接入申请

# 申请appId和appKey

  • appId： 分配给第三方应用的appId, 应用的唯一标识
  • appKey： 分配给第三方应用的appKey, 应用服务调用的密钥

appId和appKey为应用的身份认证凭证，请联系新意商务人员或售后人员申请。

新意服务开放平台提供应用的创建管理，为第三方应用生成应用的appId和appKey，并赋予应用的指定服务访问权限。

# 接入流程

  1. 通过appId和appKey等参数调用服务平台的鉴权接口，生成access_token(访问令牌)，参考：获取访问令牌接口
  2. 在access_token有效期限之内，可以访问开放平台服务

# 3.2 刷新访问令牌

第三方应用的接入到服务平台的访问令牌有一定的有效期，超过有效期的访问令牌调用接口时，会提示”已超时“，调用刷新访问令牌接口来获取新的访问令牌。

请求方法

POST

请求地址

http://host:port/EPOSG/api/1.0/authorize?sign=mysign

接口版本

1.0

请求参数

参数 类型 是否必须 描述
appid String 是 分配给应用的appId
state String 是 单笔服务调用的唯一标识，返回的报文将返回该标识
timestamp String 是 时间戳，格式为yyyyMMddHHmmssSSS，例如：20130506135203528
grant_type String 是 固定值"RT"
refresh_token String 是 刷新令牌，在首次登录时或刷新令牌时产生的刷新令牌，下次令牌超时以该令牌来获取新的访问令牌
sign String 是 请求签名，详见本文的签名规则

公共参数

调用任何一个API都必须传入的参数，目前支持的公共参数有：

# 关于签名

为了确保用户发送的请求不被恶意篡改，调用任何一个接口都需要携带签名。服务端会对根据请求参数校验签名的合法性，签名不合法的请求将被拒绝。 签名生成规则：

POST请求签名生成规则

  • 将POST请求的内容(JSON串）做MD5加密
  • 将MD5加密后的字符串进行RSA加密，以应用密码做为加密密钥，得到请求签名。

GET请求签名生成规则

  • 将GET请求参数（包括公共参数和业务参数，但除去sign参数和access_token参数），根据参数名称的ASCII码表的顺序排序。如：
  • 将排序好的参数以参数名和参数值拼接在一起
  • 将拼接好的字符串做MD5加密
  • 将MD5加密后的字符串进行RSA加密，以应用密码做为加密密钥，得到请求签名。

# 响应码

响应码是指响应体（JSON格式）中的rep_code字段，0表示成功，其他表示失败。

具体参照下表：

响应码 描述
0 成功
999100 应用尚未认证
999101 应用无服务权限
999102 应用已认证
999103 请求过多，超过限流
999104 重复的请求
999110 请求参数校验失败
999120 签名校验失败
999130 Token校验失败
999131 Token已超时
999132 当前app有效令牌数目已超过限制
999140 服务调用错误
999900 系统内部异常
