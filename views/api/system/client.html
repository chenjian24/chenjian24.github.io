客户端使用说明

# 客户端使用说明

通过系统管理服务接口的客户端，可以更方便的访问服务。并隐藏服务底层的协议、服务认证、服务治理接入等细节。

# MAVEN依赖引入

pom.xml添加系统管理API客户端依赖项, 如下：

<dependency>
    <groupId>com.shine.epframe </groupId>
    <artifactId>epframe-system-client</artifactId>
    <version>1.0.0.0</version>
</dependency>

# 集成与初始化

# 初始化方式一：工厂方式初始化

1、 业务系统启动时调用

SystemAPIFactory.init(Properties confs)

进行初始化，其中Properties配置值可以根据业务系统的实际情况从配置文件加载，或从据库加载，或从JVM环境变量中加载，配置项说明参考“初始化参数说明”。

2、 获取服务对象方式，通过

SystemAPIFactory

工厂创建，如：

UserService userService = SystemAPIFactory.getUserService();

DepartmentService deptService= SystemAPIFactory.getDepartmentService ();

……
# 初始化方式二：Spring环境下初始化

1、 业务系统的Spring Context中加载配置类

com.shine.epframe.system.api.config.SystemSpringConfiguration

，可以是编码方式注入，如：

AnnotationConfigWebApplicationContext context = ...;

context.register(com.shine.epframe.system.api.config.SystemSpringConfiguration.class);

context.refresh();

或spring XML中注入配置，如：

<bean class="com.shine.epframe.system.api.config.SystemSpringConfiguration"></bean>

2、 Spring Context中需要增加中约定的环境属性值（也就是通过@Value注解要可以获取到配置值），配置值参考“初始化参数说明”。

3、 获取服务对象方式，通过@Autowired注入API服务类，如：

@Autowired

private UserService userService;

@Autowired

private DepartmentService departmentService;

……

初始化参数说明：

参数名 可选 说明 参数值示例
dubbo.application.name 否 项目在整个分布式架构里的名称 EPSERVICERUN
dubbo.application.qos.enable 否 关闭QOS false
dubbo.application.qos.accept.foreign.ip 否 false拒绝远端主机发出的命令，只允许服务本机执行 false
dubbo.application.qos.port 否 QOS端口 2222
dubbo.application.endpoint 是 dubbo直连地址，调试时可开启此模式 rest://127.0.0.1:8080/EPSERVICERUN/rest/
dubbo.registry.address 否 Dubbo的注册中心地址 zookeeper://127.0.0.1:2181
dubbo.registry.username 否 登录用户名 guest
dubbo.registry.password 否 登录密码 guest
dubbo.registry.group 否 分组 shine
dubbo.registry.register 否 是否注册服务到注册中心 false
dubbo.registry.subscribe 否 是否从注册中心订阅服务 true
dubbo.registry.check 否 注册中心不存在时，是否报错 true
dubbo.consumer.timeout 否 消费端调用超时时间 300000
dubbo.consumer.retries 否 重试次数 0
dubbo.consumer.check 否 消费端启动时检查服务可用 false

# 接口异常规范

客户端定义了SystemAPIException异常类，所有有可能抛出业务异常的接口已显式声明此异常类。该类包含业务异常代码及异常信息，SystemAPIException.getCode()获取异常代码，SystemAPIException.getMessage()获取业务异常信息，开发过程中调用的接口如显示声明该异常需捕获异常。
